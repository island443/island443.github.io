Vision Transformer

什么是自然语言处理中的标记（token）？

在NLP领域，标记（token）是指将文本分解成更小的、有意义的单元。这个过程通常被称为“分词”（tokenization）。

标记可以是单词、符号、短语或其他任何被认为有意义的文本片段。通过将文本分解成标记，可以对这些标记进行进一步的处理和分析，例如词性标注、命名实体识别、句法分析等。

标记的例子包括：

  • 单词：例如“The”、“cat”、“sat”、“on”、“the”、“mat”等
  • 符号：例如“，”、“.”、“?”、“!”等
  • 短语：例如“New York”、“San Francisco”等

标记是 NLP 任务中的重要步骤，因为它们提供了对文本的基础理解，从而能够进行更高级的分析和处理。

将图像拆分为块 (patch)，并将这些图像块的 线性嵌入序列 作为Transformer的输入。图像块 (patches) 的处理方式同 NLP 的标记 (tokens)，故经过线性嵌入后又叫 patch token

我们将图像分割成固定大小的Patchs，线性嵌入每个展平（flattened）后的Patchs，然后再添加Position Embedding，并将生成的矢量序列送到标准的Transformer编码器。为了执行分类，我们使用标准方法向序列中添加一个额外的可学习的“分类标记”。

图像分块再嵌入(Patch Embeddings)

这transformer模型是使用的 一维嵌入序列 进行的嵌入，那么怎么把一张图处理成适合transformer维度的向量呢？

首先把图片切割成N个小块，假设图片的分辨率是（H，W）的，有C个通道，要切割成分辨率为（P，P）的小图片，那可以切割成N=(HW)/(PP)个长度的输入序列

可学习的嵌入(Learnable Embedding)

图像分为N个图像块，输入到 Transformer 编码器中就有N个向量，但该取哪一个向量用于分类预测呢？都不合适!

一个合理的做法是手动添加一个 可学习的嵌入向量 作为用于分类的类别向量,然后与其他图像块嵌入向量一起输入到 Transformer 编码器中，最后取追加的首个可学习的嵌入向量作为类别预测结果。所以，追加的首个类别向量可理解为其他N个图像块寻找的类别信息。

位置嵌入(Position Embeddings)

几种不同的位置编码方案：

1-D 位置嵌入 (1D-PE)：考虑把 2-D 图像块视为 1-D 序列

2-D 位置嵌入 (2D-PE)：考虑图像块的 2-D 位置 (x, y)

相对位置嵌入 (RPE)：考虑图像块的相对位置

ViT 采用 标准可学习/训练的 1-D 位置编码嵌入，因为尚未观察到使用更高级的 2-D-aware 位置嵌入能够带来显著的性能提升 (当然，后续的很多 ViT 变体也使用了 2-D 位置嵌入) 在输入 Transformer 编码器之前直接将图像块嵌入和位置嵌入按元素相加：

Transformer 编码器

Transformer 编码器 由交替的 多头自注意力层 和 多层感知机块 (MLP) 构成。在每个块前应用 **层归一化 (Layer Norm)**，在每个块后应用 **残差连接 (Residual Connection)**。

在 Transformer中，MAS 后跟一个 FFN (Feed-forward network)，其包含 两个 FC 层，第一个 FC 将特征从维度 D变换成 4D，第二个 FC 将特征从维度 4D 恢复成 D，中间的非线性激活函数均采用 GeLU (Gaussian Error Linear Unit，高斯误差线性单元) —这实质是一个 MLP (多层感知机与线性模型类似，区别在于 MLP 相对于 FC 层数增加且引入了非线性激活函数，例如 FC + GeLU + FC)，实现如下：

归纳偏置与混合架构

​ **归纳偏置 (Inductive bias)**：注意到，Vision Transformer 的图像特定归纳偏置比 CNN 少得多。在 CNN 中，局部性、二维邻域结构 和 平移等效性 存在于整个模型的每一层中。而在 ViT 中，只有 MLP 层是局部和平移等变的，因为自注意力层都是全局的。二维邻域结构 的使用非常谨慎：在模型开始时通过将图像切分成块，并在微调时调整不同分辨率图像的位置嵌入。此外，初始化时的位置嵌入不携带有关图像块的 2D 位置的信息，图像块之间的所有空间关系都必须从头开始学习。

​ **混合架构 (Hybrid Architecture)**：作为原始图像块的替代方案，输入序列可由 CNN 的特征图构成。在这种混合模型中，图像块嵌入投影被用在经CNN特征提取的块而非原始输入图像块。作为一种特殊情况，块的空间尺寸可以为 1×1，这意味着输入序列是通过 简单地将特征图的空间维度展平并投影到 Transformer 维度 获得的。然后，如上所述添加了分类输入嵌入和位置嵌入，再将三者组成的整体馈入 Transformer 编码器。简单来说，就是先用 CNN 提取图像特征，然后由 CNN 提取的特征图构成图像块嵌入。由于 CNN 已经将图像降采样了，所以块尺寸可为 1×1

微调及更高分辨率

我们在大型数据集上训练VIT，但将其应用到小的下游任务时需要微调一下VIT模型。

移除了预训练的预测头部，换为一个零值初始化的 D×K 前馈层 / 全连接层，其中K是下游任务的类别数

用比预训练时更高的图像分辨率进行微调通常更有益。当提供更高分辨率的图像时，需保持图像 patchs 大小相同，此时有效图像 patchs 数变多，从而有效序列长度会变长。Vision Transformer 可处理任意序列长度 ，但 预训练的位置嵌入 (pos_embed) 可能不再有意义，因为当前的位置嵌入无法与之一一对应了。因此，根据它们在原图中的位置，对预训练的位置嵌入执行 2D 插值，以扩展微调尺寸。

超参数

如下的 ViT 主要超参数 直接影响模型参数及计算量：

Layers：Encoder Block 数量
Hidden Size D：隐藏层特征大小，其在各 Encoder Block 保持一致
MLP Size：MLP 特征大小，通常设为 4D
Heads：MSA 中的 heads 数量
Patch Size：模型输入的 Patch size，ViT 中共有两个设置：14x14 和 16x16，该参数仅影响计算量

疑问

微调和插值到底是怎么一回事？

微调（Fine-tuning）是指在预训练好的模型上进行优化，以适应特定任务或领域的数据。在ViT中，微调通常是指在预训练好的ViT模型上进行优化，以适应特定图像分类任务或特定数据集。微调的过程通常包括使用特定数据集进行训练，调整模型的参数，以使模型在特定任务上表现更好。微调的过程可以是手动进行的，也可以使用自动化算法进行。

插值（Interpolation）在ViT中通常是指在处理图像数据时，对图像的像素值进行重新计算或估计的过程。插值可以用于处理不同分辨率的图像数据。在ViT中，当使用高分辨率图像进行微调时，由于图像尺寸变大，每个patch中的像素数量保持不变，但sequence lengths会相应变大。此时，为了保持模型的性能，通常会对位置嵌入向量进行插值处理，以重新计算或估计这些向量的值。插值处理可以帮助ViT模型更好地处理不同分辨率的图像数据，从而提高模型的泛化能力。

归纳偏值是什么?CNN那些特性是什么？有什么用？

在ViT（Vision Transformer）中，归纳偏置（Inductive Bias）指的是模型在处理图像数据时所采用的一组假设或规则，这些假设或规则可以帮助模型更好地泛化到未见过的数据。

在ViT中，归纳偏置通常包括以下几个方面：

  1. 图像的二维结构：ViT假设图像中不同元素之间存在二维结构，这种结构可以帮助模型捕捉到图像中的空间信息。例如，ViT中的位置嵌入向量会根据每个patch在图像中的位置进行编码，这些位置信息可以帮助模型更好地理解图像的内容。
  2. 局部性和平移等变性：ViT假设图像中不同部分的特征具有一定的局部性和平移等变性，这种假设可以帮助模型更好地捕捉到图像中的局部细节和上下文信息。例如，在ViT的MLP层中，输入数据的每个元素都会与其周围的元素进行交互，从而帮助模型更好地理解图像中的内容。
  3. 全局性：ViT假设图像中的特征具有全局性，即图像中的每个元素都与其他元素存在一定的关联。这种假设可以帮助模型更好地捕捉到图像中的全局信息，从而更好地理解图像的内容。

总之，归纳偏置是一种有意识地对模型进行约束的方式，它可以帮助模型更好地泛化到未见过的数据。在ViT中，归纳偏置通常体现在模型的位置嵌入、MLP层的局部和平移等变性以及全局性等方面。

研究混合架构是怎么一回事？

Hybrid混合模型，就是将传统CNN特征提取和Transformer进行结合

Hybrid模型的效果。通过对比发现，在训练epoch较少时Hybrid优于ViT，但当epoch增大后ViT优于Hybrid。

VIT 的Transformer encoder的数据出来，到底是怎么变成分类的那个一维向量，做什么处理？

首先出来的数据是一个[batch,batch_num+1,dim]的一个形式，然后取第0行的数据，就是一开始添加的那个可学习的向量，此时数据类型就变成了[batch,1,dim],随后把这个数据送入MLP Head进行分类操作

MLP和MLP Head是不是一个东西？ MLP Head是什么？

在Vision Transformer（ViT）中，MLP和MLP Head是不同的组件

在ViT中，MLP通常用于实现自注意力层（self-attention layer）和前馈网络（feedforward network）。

MLP Head是ViT中的一个特定组件，通常用于分类任务。它通常是一个简单的全连接层，用于将ViT的输出特征映射到目标类别的数量。
