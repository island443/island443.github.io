JavaWeb

Java web 三大组件： Servlet(处理请求), Filter（过滤拦截请求）, Listener（监听器）

Filter(过滤器)

当客户端向服务器端发送一个请求时，如果有对应的过滤器进行拦截，过滤器可以改变请求的内容、或者重新设置请求协议的相关信息等，然后再将请求发送给服务器端的Servlet进行处理。当Servlet对客户端做出响应时，过滤器同样可以进行拦截，将响应内容进行修改或者重新设置后，再响应给客户端浏览器。在上述过程中，客户端与服务器端并不需要知道过滤器的存在。

在一个Web应用程序中，可以部署多个过滤器进行拦截，这些过滤器组成了一个过滤器链。过滤器链中的每个过滤器负责特定的操作和任务，客户端的请求在这些过滤器之间传递，直到服务器端的Servlet。

Listener（监听器）

作用：在javaweb中Listener是servlet规范定义的一种规范类，主要监听3个作用域的创建，销毁以及其属性变更。

Servlet三个作用域分别为： request session ServletContext

三大域对象创建和销毁的监听器

ServletContextListener 监听ServletContext域的创建与销毁的监听器，Servlet域的生命周期：在服务器启动创建，服务器关闭时销毁

ServletContextListener监听器的主要作用：(1) 初始化的工作 初始化对象,初始化数据,比如加载数据库驱动，对连接池的初始化 (2) 加载一些初始化的配置文件；比如spring的配置文件 (3) 任务调度（定时器Timer/TimerTask）

HttpSessionListener 监听Httpsession域的创建与销毁的监听器。HttpSession对象的生命周期：第一次调用request.getSession时创建；销毁有以下几种情况（服务器关闭、session过期、 手动销毁）

ServletRequestListener 监听ServletRequest域创建与销毁的监听器。ServletRequest的生命周期：每一次请求都会创建request，请求结束则销毁。

Servlet

Servlet是三组件中最重要的一部分，但对于Servlet这里只谈谈个人理解，不谈应用

  • Servlet不能独立运行，它必须被部署到Servelt容器中，由容器来实例化和调用Servlet方法，Servlet容器在Servlet生命周期内包容和管理Servelt。

  • 用户单击某个链接或者直接在浏览器的地址栏中输入URL来访问Servlet，Web服务器接收到该请求后，并不是将请求直接交给Servlet，而是交给Servlet容器。Servlet容器实例化Servlet，调用Servlet的一个特定方法对请求进行处理，并产生一个响应。这个响应由Servlet容器返回给Web服务器，Web服务器包装这个响应，以HTTP响应的形式发送给Web浏览器。

  • Tomcat具有处理静态页面的能力，此外还是servlet和jsp的容器

  • 在servlet中，可以通过servletconfig对象获取描述servlet容器运行环境的servletcontext对象，使用该对象，servlet可以和它的servlet容器进行通信

  • servlet对象使用servletconfig对象在servlet初始化期间向它传递配置信息，一个servlet只有一个servletconfig对象

  • genericservlet有两个init（）方法，HTTPservlet有两个servlet（req，reps）方法，其中一个是将req，repe类型转化为HTTPreq，HTTPreps

服务器向客户端输出数据两种方式：字节流，字符流
  • 字节流：输出中文时没有乱码，因为字节流在输出中文时，默认采用本地编码输出（GBK），而浏览器默认也采用GBK编码显示，二者编码一致，所以不会出现乱码。

​ OutputStream os=response.getOutputStream（）；

​ os.write（“Hello，World”.getBytes（））；

  • 字符流：浏览器默认输出数据采用ISO-8859-1，不支持中文

​ printWrite out=response.getWriter（）；

​ out.write（“Hello World”）;

改变服务器输出数据默认编码：response.setCharacterencoding(“GBK”)

可不写因为服务器输出数据时只需设置response.setContentType(“text/html;charset=编码”)

就不会出现乱码问题，因为这条语句有以下2个功能：

1.告知浏览器用什么编码方式显示内容

2.告知服务器用什么编码方式输出数据给浏览器

还可通过设置头信息解决乱码问题

response.setHeader(“ContextType”,”text/html;charset=编码”)
